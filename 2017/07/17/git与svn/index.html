git与svn

Jul 17, 2017 | 阅读

下面是git与svn最相异的地方

  • git 分布式版本控制 基于每个人本地分支进行修改，本地merge后提交到主分支，没有网络可以提交代码到本地仓库
  • svn 基于中央仓库，没有本地仓库，所有修改都作用在外部仓库，没有网络无法提交代码
    *

下面是git在使用时的一些细节

  • git clone 可以将外部仓库的代码clone下来
  • git branch 查看分支
  • git branch name 创建分支（若主干内没有任何代码，则会创建分支失败）
  • git checkout name 进入名为name的分治内
  • git add 将修改文件加入此次要提交的内容中
  • git commit -m “注解” 提交代码到本地，””内写对于本次提交的描述
  • git push 将本地代码提交到远程仓库
